Lunar Lander Offline RL

Investigating the impact of demonstration data quality on Decision Transformer

Abstract

This experiment evaluates how the quality of demonstration data affects Offline Reinforcement Learning agents, comparing the MLP-based Conservative Q-Learning (CQL) and the Sequence-based Decision Transformer (DT) algorithms. Unlike standard Reinforcement Learning which learns by trial and error, Offline RL learns exclusively from a fixed dataset. We compared three data levels: Expert (fully trained), Medium (partially trained), and Random (untrained).

Conclusion

The results provide a clear demonstration of the architectural trade-offs between Decision Transformers (DT) and Conservative Q-Learning (CQL) across varying data qualities:

  • Peak Performance on Expert Data: The Decision Transformer achieved the highest overall score (213.4 with an 84% success rate), slightly edging out the CQL model. This highlights DT's strength in sequence modeling—when provided with excellent demonstrations, it can seamlessly mimic and reproduce optimal behavior.
  • Vulnerability to Sub-Optimal Data: Conversely, DT performed noticeably worse than CQL on the Medium dataset (-295.9 vs -181.6). Because DT casts RL as a supervised prediction problem, its performance is tightly bounded by the quality of the sequences in the dataset. It cannot easily "invent" new, better behaviors from noisy data.
  • Robustness of Q-Learning: CQL proved much more robust when dealing with limited or poor-quality transitions. By leveraging temporal difference (TD) learning and dynamic programming, CQL is able to "stitch together" scattered good actions from sub-optimal data, acting as a lower bound for expected performance.

Ultimately, while Decision Transformers demonstrate an impressive ability to surpass classical algorithms when given abundant expert data, traditional offline RL algorithms like CQL remain the safer choice for highly noisy, mixed, or heavily sub-optimal datasets.

Appendix: Experiment Setup

1. Environment

The experiment was conducted using Gymnasium's LunarLander-v3. This is a classic "box2d" control task where the agent must land a craft on a designated pad using three engines (Main, Left, Right).

2. Dataset Generation

Three datasets were generated using Stable-Baselines3 (PPO) to simulate different levels of expertise:

  • Expert: PPO agent trained to convergence (500,000 steps, average reward > 200).
  • Medium: A snapshot of the PPO agent's behavior at only 50,000 steps (early learning phase).
  • Random: Actions sampled uniformly from the discrete action space.

Each dataset contains exactly 50,000 transitions (state, action, reward, next state).

3. Training Details

The offline agents were trained using the Conservative Q-Learning (CQL) and Decision Transformer (DT) implementation from the d3rlpy library.

  • Steps: 30,000 gradient steps per model.
  • Architecture: DiscreteDecisionTransformer (context_size=20, max_timestep=1000) vs DiscreteCQLConfig (MLP default).
  • Hardware: Training performed on GPU (NVIDIA RTX 2080 via CUDA 12.1) to accelerate Transformer sequence modeling.

4. Evaluation Methodology

Each trained model was evaluated over 50 test episodes. Success was defined as a final cumulative reward ≥ 200. The simulation videos include a boundary check: if the agent flies significantly off-screen (e.g., y > 1.2 or |x| > 1.1), the episode is terminated early to keep the visualization focused on the landing area.
